## Context — why this change

The finished pipeline (`plan/03`) built an eGRN for a **single** cell type in a **single** region
(`EN-L2_3-IT` ∩ `Region==V1`, 5,558 cells). Because only one cell type was present, the model had **no
cell-type contrast** — which is exactly why layer-identity TFs like **CUX2 and RORB never formed
cistromes/eRegulons**, and why regional (PFC↔V1) regulation — the paper's central theme — could not be
studied at all.

This plan scales the run to **all four layer-resolved IT excitatory types across both cortical regions and
all developmental stages**, so the eGRN captures layer-identity and areal-identity regulatory programs and
their developmental dynamics in one unified model.

**New scope:** `Type ∈ {EN-L2_3-IT, EN-L4-IT, EN-L5-IT, EN-L6-IT}` **and** `Region ∈ {PFC, V1}`, **all 5
`Group` stages** → **49,283 cells** (~9× the 5,558-cell run). The tiny `General` region pool (439 IT cells)
and unlayered `EN-IT-Immature` are excluded (user decision: layer-resolved IT only).

Verified population sizes (from `GEX_with_raw_counts.h5ad` obs):

| Type | PFC | V1 | Type total |
| --- | ---: | ---: | ---: |
| EN-L2_3-IT | 6,292 | 5,558 | 11,850 |
| EN-L4-IT | 8,239 | 11,834 | 20,073 |
| EN-L5-IT | 6,069 | 3,552 | 9,621 |
| EN-L6-IT | 5,335 | 2,404 | 7,739 |
| **Region total** | **25,935** | **23,348** | **49,283** |

`Group` levels: First_trimester, Second_trimester, Third_trimester, Infancy, Adolescence. Strong stage×region
imbalance persists (V1 postnatal-dominated; PFC carries more prenatal cells) — matters for Stage 6.

## User-confirmed design decisions

1. **Scope** = the 4 layer-resolved IT types, PFC+V1, all stages (49,283 cells). Exclude General + Immature.
2. **Peak calling** = **pooled single MACS3 call** over all IT barcodes (same as the L2/3 run, just more
   barcodes) → `get_consensus_peaks` (single-entry dict, unchanged `12b` logic). *No per-population
   pseudobulk.*
3. **Region sets** = **LDA topics only** (`binarize_topics`, `ntop=3000`). *No DAR / `find_diff_features`
   step* — keeps Stage 3c structurally identical to the L2/3 run.

These choices keep the pipeline mechanically close to `plan/03`; the real work is scope, renaming/isolation,
compute scaling, and QC.

**Script naming (user rule):** every new script is a **fresh file marked `.v2.`** — e.g.
`10.v2.select_it.py`, `12.v2.call_peaks_it.sh`. **Do not modify any finished L2/3 script** (`10_select_l23.py`,
`12_call_peaks.sh`, `14_run_pycistopic.py`, `18_qc.py`, …); the `.v2.` files are new copies adapted to the IT
scope. This includes `19_organize_regulons.py` → a `.v2.` copy repointed at `work_it/` (leave the original as-is).

### Stage 3c — cisTopic + LDA (`scripts/14.v2.run_pycistopic_it.py`) — **topics only, no DAR**
Structurally identical to `14_run_pycistopic.py`; **keep the DAR/`find_diff_features` step omitted** (user
decision). Changes are paths, `PROJECT="it"`, and scaled LDA params:
- **`N_TOPICS = [20, 30, 40]`** (reduced sweep, user request; cf. the L2/3 run's `[10,20,30,40]`).
  Auto-select by coherence (`Minmo_2011`); if the curve is still rising at 40, extend upward in a follow-up.
- **`N_ITER = 150`** (keep); **`N_CPU_CISTOPIC = 1`** (keep serial join — the ray parallel-join NaN bites
  *harder* at scale). Serial region×fragment join now ≈ **30 min–2 h** on the ~14 GB IT fragment file
  (runtime, not correctness).
- **`N_CPU` for `run_cgs_models`**: parallelizes across the topic list (one model per worker). Set
  `n_cpu=min(len(N_TOPICS), cores)`. **Memory is the limiter** — each worker holds the ~49,283×~150–200k
  binary matrix + topic tables; if RAM-constrained, run the sweep in two batches rather than risk OOM.
- Binarize topics (`method="ntop", ntop=3000`) → `${REGION_SETS_DIR}/topics_top_3k/` (the *only* region-set
  subfolder). Save `${CISTOPIC_OBJ}`.

### Stage 5 — GRN inference (`scripts/17.v2.run_grn_it.sh`) — same CLI, scaled compute

| Step | 5,558-cell | ~49k expectation |
| --- | --- | --- |

### Stage 6 — QC redesign (`scripts/18.v2.qc_it.py`) — the scientific payoff
Rework from "single cell type → stage dynamics only" to a **3-way Type × Region × Group** analysis. Attach
the 3 obs columns from `${IT_GEX_H5AD}` to the AUCell matrix (strip the `___it` suffix; the existing
barcode-reindex + fail-fast-on-NaN pattern carries over).

1. **Layer-specific positive controls** (replace the flat `KNOWN_L23_TFS`):
   - L2/3: CUX2, POU3F2 (BRN2), POU3F3, SATB2, MEF2C
   - L4: RORB, MEF2C, RFX3
   - L5: FEZF2, BCL11B (CTIP2), ETV1, FOXP2, SOX5
   - L6: TBR1, FOXP2, TLE4, SOX5, FOXP1, NR4A2
   Check each layer-marker regulon peaks in its own layer (AUCell grouped by `Type`). **Explicitly report
   whether CUX2/RORB now form eRegulons** — the key expected benefit of adding cell-type contrast.
2. **Region-specific activity (PFC vs V1)** — the paper's theme: differential AUCell PFC-vs-V1 **within
   matched (Type, Group)** strata (control for layer + stage given the imbalance) → V1-biased vs PFC-biased
   regulons (candidates: NR2F1/COUP-TFI, PBX3). Heatmap of region-differential activity.
3. **Full 3-way readouts:** (a) regulon×Type heatmap (layer identity); (b) PFC-vs-V1 within matched
   Type/Group; (c) developmental trajectories faceted by Type×Region (generalizes the single-track stage
   plot); (d) **per-stratum N reported alongside every activity estimate** so region effects aren't
   confounded by stage composition.
