# Executable Plan: SCENIC+ eGRN for Wang25 L2/3 (`EN-L2_3-IT`) neurons

> **Status: executable.** Concrete paths, verified CLI, and params. Supersedes the skeleton
> `plan/02.wang25_l23_scenicplus_skeleton.md` (all three open questions resolved there).

## Goal

Infer an enhancer-driven GRN (eRegulons) for **V1 L2/3 IT excitatory neurons**
(`Type == "EN-L2_3-IT"` **and** `Region == "V1"`, **5,558 cells**) from the paired 10x snMultiome of
**Wang L. et al., *Nature* 647:169–178 (2025)**, "Molecular and cellular dynamics of the developing
human neocortex" (hg38).

## Resolved decisions (from the skeleton)

- **Data = paired 10x snMultiome** (RNA+ATAC same nuclei), CELLxGENE deposit. RNA↔ATAC barcodes verified
  identical (`{Sample_ID}_{16bp}-1`). Label column = `obs["Type"]`, region column = `obs["Region"]`.
- **Scope (Q3) = `EN-L2_3-IT` in `Region == "V1"` only → 5,558 cells.** PFC (6,292) and the tiny
  "General" fetal pool (78) are **excluded**; immature/newborn types excluded. Rationale: a single-region
  (V1) eGRN avoids the PFC↔V1 regional-regulatory confound (a central theme of the paper) and the
  age/region imbalance seen when pooling. Stage distribution of the V1 set:

  | Region | 1st trim | 2nd trim | 3rd trim | Infancy | Adolescence | **Total** |
  | --- | ---: | ---: | ---: | ---: | ---: | ---: |
  | General | 60 | 18 | 0 | 0 | 0 | 78 |
  | PFC | 0 | 28 | 494 | 4,011 | 1,759 | 6,292 |
  | **V1 (used)** | **0** | **38** | **108** | **3,672** | **1,740** | **5,558** |
  | Total | 60 | 84 | 602 | 7,683 | 3,499 | 11,928 |

  V1 L2/3 is ~94% postnatal (Infancy 3,672 + Adolescence 1,740); the ~146 prenatal V1 cells blend in.
- **cisTarget DB (Q2) = custom-built** on the V1 L2/3 consensus peaks (not a prebuilt screen DB).

## ⚠️ Chromosome naming convention — UCSC (`chr1`) everywhere (CRITICAL)

**Verified at runtime (2026-07-14)** — the two inputs disagree in *different* ways, confirmed by
`tabix -l` on the raw fragment index and by inspecting the raw biomart output:
- **ATAC fragments are MIXED**: primary chroms are **already UCSC** (`chr1`–`chr22`, `chrX`, `chrY`);
  only the **scaffolds are Ensembl-named** (`GL000009.2`, `KI270711.1`, …); there is **no `chrM`**.
  (An early head-of-file peek showed only `GL000009.2` rows — scaffolds sort first — which misleadingly
  looked all-Ensembl; the index listing is authoritative.)
- **Gene annotation is fully Ensembl-style** (`1`, `MT`, `GL000009.2`, `HG1012_PATCH`): biomart only
  converts to UCSC when NCBI is reachable — it was not, so it fell back.
- **Fasta / chrom sizes / blacklist are UCSC** (`chr1`, …) with UCSC-named scaffolds (`chrUn_GL...`).

A silent mismatch yields **empty region↔gene / region↔fragment overlaps → an empty GRN with no error**.
Decision: **standardize on UCSC, primary chromosomes only** (`chr1`–`chr22`, `chrX`, `chrY`, `chrM`;
drop all scaffolds/patches/alts — the Ensembl-named fragment scaffolds aren't in our UCSC chromsizes anyway).

Concretely:
- **Gene annotation** (Stage 2): biomart output written to `${GENE_ANNOT_RAW}` (Ensembl), then normalized
  by `awk` to `${GENE_ANNOT}` — `1..22/X/Y → chr*`, `MT → chrM`, scaffolds/patches dropped.
  Result: 103,810 rows on 25 chroms (from 112,474 raw).
- **Fragments** (new **Stage 2b**, `scripts/11b_fragments_to_ucsc.sh`): primary chroms need **no
  renaming** — just **drop the Ensembl-named scaffolds**. Stream the 40 GB `${FRAG_BGZ}`, keep rows on
  `chr1..chr22/chrX/chrY` → `${FRAG_UCSC_BGZ}` (bgzip) + tabix. All of Stage 3 (peak calling **and**
  `create_cistopic_object_from_fragments`) reads `${FRAG_UCSC_BGZ}`, **not** raw `${FRAG_BGZ}`.
- **Fasta / chrom sizes / blacklist**: already UCSC — used as-is.

This **resolves open risk #3**.

## Stage 2b — Restrict fragments to primary UCSC chroms (`scripts/11b_fragments_to_ucsc.sh`)

Runs once after the Stage 0 download completes. Primary chroms are already UCSC, so this only **drops the
Ensembl-named scaffolds** (no renaming), then re-indexes. Output feeds all of Stage 3.
`bgzip`/`tabix` come from the **chrombpnet** env (absent in `epics`).

```bash
BGZIP=/home/qlyu/miniforge3/envs/chrombpnet/bin/bgzip
TABIX=/home/qlyu/miniforge3/envs/chrombpnet/bin/tabix
zcat ${FRAG_BGZ} \
  | awk -F'\t' 'BEGIN{OFS="\t";
                split("chr1 chr2 chr3 chr4 chr5 chr6 chr7 chr8 chr9 chr10 chr11 chr12 chr13 chr14 chr15 chr16 chr17 chr18 chr19 chr20 chr21 chr22 chrX chrY chrM",a," ");
                for(i in a) keep[a[i]]=1}
               /^#/{print; next}
               $1 in keep' \
  | ${BGZIP} -c > ${FRAG_UCSC_BGZ}
${TABIX} -p bed ${FRAG_UCSC_BGZ}      # rows stay chrom-contiguous + position-sorted → tabix-valid
```

**Check:** `${TABIX} -l ${FRAG_UCSC_BGZ}` lists only `chr1..chr22, chrX, chrY`; barcodes unchanged
(`{Sample_ID}_{16bp}-1`).

## Stage 3 — pycisTopic + custom cisTarget DB

### 3a. Peak calling (MACS3) on L2/3 fragments (`scripts/12_call_peaks.sh`) — mirrors v1astro `10.call_peaks_macs3.sh`

> **Peak-set decision (2026-07-14): call our own L2/3 peaks — not Wang25's reported set.**
> Investigated using the authors' peaks: no standalone peak BED is published. Their peaks live only in
> the 32.8 GB atlas Seurat `.rds` (needs R/Signac) or the 3.42 GB Cellranger `filtered_feature_bc_matrix.h5.zip`
> (Python-extractable), and both are a **global atlas** set (all cell types, PFC+V1, all ages). For a
> single-cell-type L2/3 eGRN, cell-type pseudobulk MACS peaks are the SCENIC+-recommended input and avoid
> diluting the cisTarget DB / topic model with regions inaccessible in L2/3. Authors' peaks may later be
> fetched only as an overlap **cross-check**, not as the input. (Data: Dryad doi:10.5061/dryad.2280gb612.)

1. Subset `${FRAG_UCSC_BGZ}` (UCSC, primary chroms — Stage 2b) to L2/3 barcodes → BED of fragment
   intervals (filter col4 ∈ `L23_BARCODES`).
2. `macs3 callpeak -f BED -t <l23.bed> -g hs -n l23 --outdir ${WORK} --nomodel --shift -75 --extsize 150
   -q 0.01 --call-summits`  ← **`-g hs`** (human; mouse used `-g mm`). → `l23_peaks.narrowPeak` (209,551 peaks).
3. **Consensus peaks (`scripts/12b_consensus_peaks.py`, epics env)** — `pycisTopic.get_consensus_peaks`
   re-centers each summit to fixed 500 bp and iteratively drops less-significant overlapping peaks
   (Corces et al. 2018), applying `${HG38_BLACKLIST}` → `${PEAKS_FILTERED}`. This replaces the naive
   "extend every summit ±250" (which left 208,641 heavily overlapping windows). **Result: 141,169
   non-overlapping 500 bp peaks** (verified merged==count). Overlapping set kept as
   `l23_summits_501_filtered.overlapping.bed` for provenance.

### 3c-prep. Extract L2/3-only fragments (`scripts/14a_extract_l23_fragments.sh`)

`create_cistopic_object_from_fragments` reads the ENTIRE fragment file (all ~243k nuclei) before
filtering by `valid_bc` — a ~30-min read of the 40 GB file per attempt. Pre-filter once to the 5,558
L2/3 barcodes (+ hard-validate 5 cols / integer count) → `${L23_FRAG_GZ}` (**1.6 GB**, 130,965,127 rows,
tabix'd). Confirmed **0 malformed rows**. Stage 3c reads this; re-runs take minutes.

### 3c. cisTopic object + LDA topics (`scripts/14_run_pycistopic.py`) — mirrors v1astro `11.run_pycistopic.py`

> **Two runtime gotchas (both resolved 2026-07-15):**
> 1. **`.gz` extension required** — pycisTopic `gzip.open`s a fragment file only if its name ends in
>    `.gz` (else reads bytes as text → `UnicodeDecodeError`). Our Stage 2b output is named `.gz`.
> 2. **`n_cpu=1` for the cisTopic build** — with `n_cpu>1`, `regions.join(fragments)` uses ray and at
>    full scale returns NaN in the fragment-count column → `Score.astype(int32)` `IntCastingNaNError`.
>    The serial join (n_cpu=1) is correct; LDA below keeps `n_cpu=8`.

```python
from pycisTopic.cistopic_class import create_cistopic_object_from_fragments
from pycisTopic.lda_models import run_cgs_models, evaluate_models
from pycisTopic.topic_binarization import binarize_topics

cistopic_obj = create_cistopic_object_from_fragments(
    path_to_fragments=L23_FRAG_GZ, path_to_regions=PEAKS_FILTERED,   # L2/3-only fragments (3c-prep)
    path_to_blacklist=HG38_BLACKLIST, project="l23",
    valid_bc=open(L23_BARCODES).read().split(),   # restrict to L2/3 barcodes
    n_cpu=1,                                       # serial join — avoids the ray parallel-join NaN
)   # do NOT pass split_pattern (creates duplicate cells — v1astro note)
# → ATAC binary matrix (141,169 regions × 5,558 cells)
models = run_cgs_models(cistopic_obj, n_topics=[10,20,30,40], n_cpu=8, n_iter=150,
                        random_state=555, alpha=50, alpha_by_topic=True, eta=0.1)
cistopic_obj.add_LDA_model(evaluate_models(models, select_model=None, return_model=True))
# binarize → per-topic region sets (BED) into REGION_SETS_DIR/topics_top_3k/
region_bin = binarize_topics(cistopic_obj, method="ntop", ntop=3000)
```

- **N_TOPICS**: sweep `[10,20,30,40]`, auto-select by coherence (v1astro used `[10,20,30]`, n_iter=10;
  bump `n_iter` to ~150 for a real run). Save `${CISTOPIC_OBJ}`.
- Write per-topic BEDs to `${REGION_SETS_DIR}/topics_top_3k/` (input to `motif_enrichment_cistarget`).

## Stage 4 — SCENIC+ prepare_data + motif enrichment (`scripts/16_prepare_menr.sh`)

Four steps: **4a** prepare_GEX_ACC → MuData, **4b** search_spance, **4c** motif_enrichment_cistarget,
**4d** prepare_menr → cistromes. (The skeleton omitted 4d; the eGRN's `--cistromes_fname` needs
`prepare_menr` output, not the raw cistarget result.) Verified end-to-end 2026-07-15.

> **Runtime fixes (all resolved):**
> - **4a is multiome/per-cell** — the `--key_to_group_by`/`--nr_cells_per_metacells` flags apply ONLY to
>   non-multiome mode; for paired multiome they're ignored. Link RNA↔ATAC by barcode via
>   `--bc_transform_func "lambda x: x + '___l23'"` (cisTopic suffixes ATAC barcodes with the project).
>   Add `--do_not_use_raw_for_GEX_anndata` (Stage 1 put counts in `.X`, cleared `.raw`).
> - **RNA must use gene SYMBOLS** — CELLxGENE ships Ensembl IDs as `var_names`; Stage 1 now remaps to
>   `feature_name` symbols (else 4b `search_spance` → `KeyError: 'Gene'`, 0 gene↔annotation overlap).
> - **4b chromsizes format** — `search_spance` reads chromsizes via `pd.read_table` expecting a
>   `Chromosome/Start/End` header (BED-like), NOT the 2-col `chrom.sizes` (which Stage 3b bedtools needs).
>   The script builds `${HG38_CHROMSIZES_SP}` inline. Set explicit `--upstream 1000 150000 --downstream 1000 150000`.
> - **4c `--region_set_folder` = the PARENT `region_sets/`** — it iterates SUBFOLDERS and reads their
>   `.bed`s (keys `topics_top_3k_TopicN`). Pointing at the `topics_top_3k` leaf → 0 sets, empty output.

```bash
SP="conda run --no-capture-output -n epics scenicplus"
${SP} prepare_data prepare_GEX_ACC --cisTopic_obj_fname ${CISTOPIC_OBJ} \
    --GEX_anndata_fname ${L23_GEX_H5AD} --out_file ${MUDATA} \
    --do_not_use_raw_for_GEX_anndata --bc_transform_func "lambda x: x + '___l23'"
${SP} prepare_data search_spance --multiome_mudata_fname ${MUDATA} \
    --gene_annotation_fname ${GENE_ANNOT} --chromsizes_fname ${HG38_CHROMSIZES_SP} \
    --out_fname ${SEARCH_SPACE} --upstream 1000 150000 --downstream 1000 150000
${SP} grn_inference motif_enrichment_cistarget --region_set_folder ${REGION_SETS_DIR} \
    --cistarget_db_fname ${CTX_RANKINGS} --output_fname_cistarget_result ${CTX_RESULT} \
    --temp_dir ${WORK}/tmp --species homo_sapiens \
    --path_to_motif_annotations ${MOTIF_TBL} --annotation_version v10nr_clust --n_cpu 8
${SP} prepare_data prepare_menr --paths_to_motif_enrichment_results ${CTX_RESULT} \
    --multiome_mudata_fname ${MUDATA} --out_file_tf_names ${TF_NAMES_MENR} \
    --out_file_direct_annotation ${CISTROMES_DIRECT} --out_file_extended_annotation ${CISTROMES_EXTENDED} \
    --direct_annotation Direct_annot --extended_annotation Orthology_annot
```

**Results:** MuData 5,558 × (35,468 genes + 141,169 regions); search_space 18 MB; ctx_results 291 MB;
**756 cistrome TFs** (`${TF_NAMES_MENR}`). Positive-control cistromes present: SATB2, POU3F2, MEF2C, FOXP1;
**absent: CUX2, RORB** (motifs not enriched in L2/3 topics → cannot form eRegulons — report in Stage 6).
Species = `homo_sapiens`; motif annotation = the **hgnc** tbl. Optionally also run DEM and merge cistromes.

## Stage 5 — GRN inference (`scripts/17_run_grn.sh`) — verified CLI from the epics Tier-2 test

```bash

${SP} grn_inference TF_to_gene \
    --multiome_mudata_fname ${MUDATA} --tf_names ${TF_NAMES_MENR} \

    --multiome_mudata_fname ${MUDATA} --search_space_fname ${SEARCH_SPACE} \
    --temp_dir ${WORK}/tmp --out_region_to_gene_adjacencies ${WORK}/region_to_gene_adj.tsv \

${SP} grn_inference eGRN \
    --TF_to_gene_adj_fname ${WORK}/tf_to_gene_adj.tsv \
    --region_to_gene_adj_fname ${WORK}/region_to_gene_adj.tsv \
    --cistromes_fname ${CISTROMES_DIRECT} --ranking_db_fname ${CTX_RANKINGS} \
    --eRegulon_out_fname ${EREGULON_OUT} \
    --temp_dir ${WORK}/tmp --gsea_n_perm 1000 --n_cpu 8 --seed 666
```
