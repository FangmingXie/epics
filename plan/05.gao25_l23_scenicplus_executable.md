# Executable Plan: SCENIC+ eGRN for gao25 (Allen DevVIS) mouse L2/3 IT CTX Glut neurons

> **Status: executable.** Adapts the human `plan/03.wang25_l23_scenicplus_executable.md` (and the more
> mature `.v2.*` all-IT scripts) to the **mouse** Allen Developmental Mouse Atlas (DevVIS) multiome.
> On first execution, save a copy under `plan/05.gao25_l23_scenicplus_executable.md` (CLAUDE.md rule).

## Context — why this plan

We want an enhancer-driven GRN (eRegulons) for **upper-layer cortical L2/3 IT excitatory neurons** in the
**developing mouse visual cortex**, mirroring the established human wang25 L2/3 pipeline so the two are
directly comparable. The paired 10x snMultiome (RNA + ATAC, same nuclei) comes from the Allen
**DevVIS** deposit. The wang25 pipeline (`scripts/*.v2.*`, human hg38) is a working, verified template;
this plan forks it with the **three requested differences** made explicit below.

## The three key differences vs. the wang25 plan (requested)

1. **Mouse, not human.** Assembly is **mm10 (GRCm38)** — *verified* (chr1 max peak end 195,233,066 fits
   mm10 chr1 195,471,971 but **exceeds** mm39 chr1 195,154,279; chrX likewise). All downstream refs are
   mm10: `--species mus_musculus`, the **MGI** motif-annotation table, a **GRCm38** gene annotation, mm10
   fasta/chromsizes/blacklist, and **mouse mixed-case gene symbols** (`Cux2`, `Satb2`, …).
2. **Peak set is precomputed** → the ATAC h5ad ships a **peaks × cells count matrix** (882,075 peaks), not
   fragments. So we **DROP** all fragment/peak-calling stages of wang25 (11b frags→UCSC, 12 MACS3, 12b
   consensus, 14a fragment-extract). cisTopic is built **from the count matrix** via
   `pycisTopic.cistopic_class.create_cistopic_object`; the custom cisTarget DB is built directly on the
   (L2/3-accessible) precomputed peak BED.
3. **Transcriptome already downloaded — YES, present.** Both multiome h5ads are on disk (Stage 0 done):
   - `DevVIS_multiome_snATAC_processed.h5ad` (45 G) — ATAC peak×cell matrix.
   - `DevVIS_multiome_snRNA_processed.h5ad` (**5.6 G — the transcriptome, present ✓**).
   - (`DevVIS_scRNA_processed.h5ad`, 38 G, is a *separate* non-multiome scRNA — **not used**.)
   **No download step needed.**

## Two scope decisions (confirmed with user)

- **Cells = cortical/VIS L2/3 only.** `subclass_label == "L2/3 IT CTX Glut"` **and** `roi ∈` the five
  cortical/visual ROIs (VIS, VIS-VISam-VISpm, VIS-VISal-VISpl-VISl-VISli-VISpor, VISp, Isocortex).
  **Drop** the non-cortical dissection tail (TH 1,305 + CH-BS 42). ≈ **21,324 ATAC** L2/3 cells before
  the RNA∩ATAC intersection; region kept as a QC covariate.
- **Peaks = filter to L2/3-accessible.** From the 882,075 precomputed peaks, keep those accessible in
  ≥ `MIN_CELL` L2/3 cells (default **≥1% of cells**), then use that reduced set for **both** the cisTopic
  model and the cisTarget DB (lighter build, cell-type-relevant; matches wang25's "don't dilute the DB").

## Data characterization (verified this session)

**ATAC** `DevVIS_multiome_snATAC_processed.h5ad` — AnnData, CSR in `X` (int counts), **194,840 cells ×
882,075 peaks**. `var/_index` = `chr1_3079841_3080341` (underscore, 500 bp fixed windows; chr1–19 + chrX,
no Y/M). `obsm/uns/layers` empty. obs has full label hierarchy + `age_label`, `roi`, `sex`, `donor_name`,
`library_prep`, `sample_id`. (See `~/mydata/data/gao25/orig/README.atac.txt`.)

**RNA** `DevVIS_multiome_snRNA_processed.h5ad` — AnnData, CSR in `X` (**int32 raw counts** — exactly what
SCENIC+ needs; no `.raw`/layers), **200,061 cells × 32,285 genes**. `var/_index` = **mouse gene symbols**
(`Xkr4`, `Sox17`, …). Same label hierarchy as ATAC.

**L2/3 IT CTX Glut counts:** ATAC 22,671 / RNA 22,705 total; cortical/VIS-only ≈ 21,324 (ATAC). By
`age_label`: negligible embryonic (E15.5–E18 ≈ 44), postnatal-dominant, peaks at **P14** (~5.5k) and
adult **P56** (~6.4k); present at 13/14 timepoints (absent E16).

**RNA ↔ ATAC barcode correspondence (linchpin — the two files swap the same two conventions):**

| key | RNA column | ATAC column |
| --- | --- | --- |
| `{well}_MTX-{n}_mouse#{16bp}-1` | `obs/cellNames` | `obs/_index` |
| `{16bp}-{library_prep}-{numID}` | `obs/_index` | `obs/sample_id` |

→ **Multiome pairing key = RNA `cellNames` ≡ ATAC `_index`.** We standardize *both* modalities' cell
names to this string, so cisTopic (built from ATAC, cells named by ATAC `_index`) and the RNA h5ad
(obs_names ← `cellNames`) reconcile after the `___gao25l23` suffix. Final multiome set = **RNA∩ATAC**
intersection of the cortical L2/3 barcodes (computed at runtime, ≈ 21k — do **not** hardcode; assert > 15k).

## Stage 2 — Mouse references (`scripts/11.gao25.refs.sh`)

mm10 fasta/chromsizes/blacklist are reused in place (no copy needed). Only the **gene annotation** must be
produced, and it **must be GRCm38** to match the mm10 peaks (Ensembl's current default is GRCm39 → would
give mismatched coordinates → empty region↔gene search space → silent empty GRN).

```bash
mkdir -p ${DB}
# GRCm38 gene annotation via the last GRCm38 Ensembl archive host (release 102, Nov 2020)
conda run --no-capture-output -n epics scenicplus prepare_data download_genome_annotations \
  --species mmusculus \
  --biomart_host http://nov2020.archive.ensembl.org \
  --genome_annotation_out_fname ${GENE_ANNOT_RAW} \
  --chromsizes_out_fname ${DB}/mm10_chromsizes_ucsc.tsv
# normalize Ensembl→UCSC: 1..19/X/Y → chr*, MT → chrM, drop scaffolds/patches → ${GENE_ANNOT}
awk 'BEGIN{OFS="\t"} NR==1{print;next}
     { c=$? ; ... map col holding Chromosome: /^([0-9]+|[XY])$/→"chr"$col; "MT"→"chrM"; else skip }' ...
# search_spance chromsizes (Chromosome/Start/End header) from mm10.chrom.sizes primary chroms (chr1-19,X,Y,M)
printf 'Chromosome\tStart\tEnd\n' > ${MM10_CHROMSIZES_SP}
awk 'BEGIN{OFS="\t"} $1 ~ /^chr([0-9]+|[XYM])$/ {print $1,0,$2}' ${MM10_CHROMSIZES} >> ${MM10_CHROMSIZES_SP}
```

- **Verify at runtime:** confirm `download_genome_annotations` accepts `--biomart_host` in SCENIC+ 1.0a2
  (`scenicplus prepare_data download_genome_annotations --help`). **Fallback** if not: derive TSS/gene-body
  TSV from a **GENCODE vM23 / Ensembl r102 GRCm38 GTF** (columns SCENIC+ expects: Chromosome, Start, End,
  Strand, Gene, Transcript_type). Either way the output must be **UCSC `chr` names, GRCm38 coords**.
- Confirm the awk column index of the chromosome field from the actual biomart header before running the
  normalize step (mirror the human `11_refs.sh` normalizer; mouse has chr1–19, not 1–22).

### 3c. cisTopic object **from matrix** + LDA topics (`scripts/14.gao25.run_pycistopic.py`)

The core structural change. Build with `create_cistopic_object` (in-memory), **not** `_from_fragments`.

```python
from pycisTopic.cistopic_class import create_cistopic_object
from pycisTopic.lda_models import run_cgs_models, evaluate_models
from pycisTopic.topic_binarization import binarize_topics
import numpy as np, scipy.sparse as sp

npz = np.load(L23_MATRIX_NPZ, allow_pickle=True)
mat = sp.csr_matrix((npz["data"], npz["indices"], npz["indptr"]), shape=tuple(npz["shape"]))  # regions×cells
cistopic_obj = create_cistopic_object(
    fragment_matrix=mat,                     # regions×cells CSR
    cell_names=list(npz["cell_names"]),      # ATAC _index (common key)
    region_names=list(npz["region_names"]),  # 'chr:start-end'
    path_to_blacklist=MM10_BLACKLIST,        # drops blacklisted regions
    project=PROJECT,                          # tag_cells → cell names get '___gao25l23'
    min_frag=1, min_cell=1,                  # peak-level filtering already done in 3a
)   # split_pattern default '___' → suffix '___gao25l23' (matches Stage 4 bc_transform)
# attach cell metadata (subclass/roi/age_label/sex/donor), indexed by the tagged cell names
cistopic_obj.add_cell_data(cell_meta_df)     # index must match cistopic cell names (barcode + '___gao25l23')

models = run_cgs_models(cistopic_obj, n_topics=[10,20,30,40], n_cpu=3, n_iter=150,
                        random_state=555, alpha=50, alpha_by_topic=True, eta=0.1)
cistopic_obj.add_LDA_model(evaluate_models(models, select_model=None, return_model=True))
region_bin = binarize_topics(cistopic_obj, method="ntop", ntop=3000)   # → region_sets/topics_top_3k/*.bed
```

- No fragment-based QC (TSS/FRiP) is available from a bare matrix — rely on `min_frag/min_cell` + the h5ad
  obs QC; note this in the report. **Topics-only region sets** (no DAR), consistent with the v2 decision.
- Region-set BEDs written by splitting `chr:start-end` → `chr\tstart\tend` (as in `14.v2`).

## Open risks / verify-at-runtime

1. **GRCm38 gene annotation (HIGH).** Peaks are mm10/GRCm38; Ensembl default is now GRCm39. Must pin the
   GRCm38 archive host (or a GRCm38 GTF). A silent assembly mismatch → empty region↔gene overlaps → empty
   GRN with no error. Verify the search-space row count is non-trivial after Stage 4b.
2. **cisTopic-from-matrix metadata alignment.** `add_cell_data` index must match the tagged cell names
   (`barcode + '___gao25l23'`). Verify `cistopic_obj.cell_data` is fully populated (no all-NaN columns).
3. **Barcode reconciliation.** Confirm RNA `cellNames` values are byte-for-byte present as ATAC `_index`
   for the same nuclei (Stage 1 logs the intersection size; investigate if it collapses).
4. **DB build weight.** Even L2/3-filtered, the peak set may be a few hundred k regions × ~10k motifs —
   run `-t 20`+ cores; expect a long build (mirrors v1astro).
5. **`min_frag/min_cell` on a pre-binarized-ish matrix.** The precomputed matrix is fragment counts, not
   binary; `is_acc` binarizes internally. Sanity-check `cistopic_obj.binary_matrix` shape vs. the filtered
   peak/cell counts.
6. **Marker expectations.** Mouse Cux2 is a canonical L2/3 marker; if it fails to form a cistrome/eRegulon
   (as human CUX2 did in the L2/3-only run), note it — an all-layer contrast (future) may recover it.
