## Stage flow per subclass (identical method to L2/3)

1. **S1 select** (`10`): mask `subclass_label==<label>` ∧ `roi∈VIS_ROIS`; RNA `cellNames`∩ATAC `_index`;
   `obs_names←cellNames`; raw int counts in `.X`; assert intersection > `MIN_INTERSECT`.
2. **S3a peaks** (`12`): subset ATAC (backed) to the barcodes; keep peaks in ≥5% of cells; write BED +
   int32 regions×cells npz.
3. **S3b DB** (`13`): custom mm10 cisTarget DB on the layer peaks (48 cores; ~20 min or less).
4. **S3c cisTopic** (`14`): build from the matrix; LDA sweep `[10,20,30,40]`; binarize topics.
5. **S4 menr** (`16`): MuData + search space (reuse shared gene annot/chromsizes) + `mus_musculus`/MGI
   motif enrichment → cistromes + tf_names.
6. **S5 GRN** (`17`, `17b`): TF2G (GBM) / R2G (GBM+SR) / eGRN (direct + extended) / AUCell.
7. **S6/S7** (`18`, `19`): QC developmental cascade + layer markers; merged regulon→gene table.

## Execution order & commits
Run **sequentially, L4 → L5 → L6**; do not start the next layer until the current one is verified.
Within a layer, commit stage-by-stage (as in the L2/3 run) and copy the final
`regulon_gene_table.tsv` to `reports/gao25_<tag>_regulon_gene_table.tsv` (tracked; `data/` is gitignored).
Heavy stages (3b DB, 5 GRN) run in the background; monitor to completion before committing.

Rough cost: **l4** ≈ the L2/3 run (~2–3 h wall, mostly Stage 5 GBM); **l5/l6** faster (~1–2 h) given ⅓
the cells and a smaller peak set.

## Verification (per subclass, mirrors the L2/3 checks)
- **S1:** `vis_<tag>_gex.h5ad` obs count == `wc -l` barcodes; `.X` integer; intersection > `MIN_INTERSECT`.
- **S3a:** peaks-after-5%-filter logged; npz shape = (n_regions, n_cells); assert n_keep ≥ 10 000.
- **S3b:** both feathers exist; region count == BED lines.
- **S3c:** cisTopic `binary_matrix` shape sane; `cell_data` fully populated (no all-NaN cols); N topic BEDs.
- **S4:** search-space rows ≫ 0 (reuses the already-validated GRCm38 annotation); cistrome TFs reported;
  layer markers checked among cistromes.
- **S5:** `eRegulon.tsv` non-empty; log #eRegulons/#TFs/#triplets; `AUCell.h5mu` per-cell.
- **S6/S7:** QC CSV/PNGs; `regulon_gene_table.tsv` merged. Compare eRegulon count/markers across layers and
  to the L2/3 anchor (76 direct / 51 TFs); confirm the layer-defining TF (Rorb/Fezf2/Tbr1) surfaces.

## Risks / notes
1. **L4 ≡ L4/5** (user-confirmed): the "L4" run is the merged L4/5 IT subclass; label outputs `l4`.
2. **Smaller L5/L6 populations** (~8 k / 7.4 k): lower `MIN_INTERSECT`→5000; the 5% peak filter yields a
   smaller peak set (faster). Watch the `n_keep ≥ 10 000` assert in S3a (expected to pass comfortably).
3. **Per-layer DBs**: each subclass builds its own cisTarget DB on its own accessible peaks (consistent
   with the L2/3 "don't dilute the DB" design) — do not share one DB across layers.
4. **Marker recovery**: a TF forming a cistrome but not an eRegulon is expected (as Rorb did in L2/3);
   report both levels. The layer-defining TF surfacing as an eRegulon is the key biological sanity anchor.
