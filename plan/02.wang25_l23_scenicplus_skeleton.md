# Skeleton Plan: SCENIC+ on Wang et al. 2025 (human), L2/3 neurons only

> **Status: rough skeleton.** High-level stages + open questions to resolve before a
> detailed, executable plan is written. Not yet ready to run.

## Context / goal

Run the (now-working) SCENIC+ pipeline in the `epics` env on the epigenomics data from
**Wang et al. 2025 (Nature)** — a **developing human brain** atlas — restricted to
**L2/3 excitatory neurons**. Produce an eGRN (eRegulons) for that population.

What we know from data already on disk (`/data/qlyu/data/wang25_human/`, MERFISH portion):
- Developing human cortex; samples span 2nd trimester → infancy; regions V1/BA17/BA10/PFC/MFG.
- Cell-type annotation scheme: **L2/3 = `type == "EN-L2_3-IT"`** (22,667 cells in MERFISH),
  niche "Cortical layer 2 and 3". This is the label to select on (or its equivalent in the
  epigenomics object).

Key differences from the prior mouse pipeline (v1_astro / the `epics` test run):
- **Species = human → hg38**, human motif collection (**hgnc**), human cisTarget database.
  The proven test used mm10/mouse assets; those must all be swapped to human.

## Data access

**Paper (identified):** Wang L, Wang C, Moriano JA, Chen S, Zuo G, Cebrián-Silla A, … Kriegstein AR.
"Molecular and cellular dynamics of the developing human neocortex." *Nature* 647:169–178 (2025).
DOI 10.1038/s41586-024-08351-7 (bioRxiv 2024.01.16.575956). First author **Li Wang** → the "Wang25" tag.

Confirmed as the source of the on-disk MERFISH: uses the same annotation scheme (**Tri-IPC**,
RG-oRG/tRG, EN-Newborn, EN-IT-Immature, **EN-L2_3-IT**), and the Dryad `MERFISH_Seurat_object.rds`
is the same **6 samples** as `/data/qlyu/data/wang25_human/MERFISH_host.h5ad`.

**Chromatin modality = paired 10x snMultiome (RNA+ATAC, same nuclei)** → ideal for SCENIC+, no
label transfer needed. 38 neocortical samples (PFC + V1), 243,535 nuclei post-QC. The multiome RNA
is whole-transcriptome (feeds SCENIC+ GEX), so the 300-gene MERFISH panel is not used for GRN.

Deposition (verbatim from the paper):

> (https://assets.nemoarchive.org/dat-oiif74w). MERFISH data were deposited to the Brain Image

### ✅ Recommended download set (use these)

Take **both** modalities from the **same CELLxGENE collection** — ready-to-use, no R/Seurat step, and
(critically) both are already in the exact formats SCENIC+/pycisTopic expect:

| # | Modality | File (CELLxGENE) | Why preferred |
| --- | --- | --- | --- |
| 1 | RNA / GEX | `https://datasets.cellxgene.cziscience.com/a4310202-4dc8-4e1b-a96d-d9675f5b14d1.h5ad` | ready `.h5ad` (counts + metadata); no `10x-h5→h5ad` or `.rds` conversion |
| 2 | ATAC fragments | `https://datasets.cellxgene.cziscience.com/c9ba15d8-45a1-4f68-ac91-a59ad2d44a2d-fragment.tsv.bgz` | already **bgzip**-compressed = the format pycisTopic reads directly |
| 3 | ATAC index | `https://datasets.cellxgene.cziscience.com/c9ba15d8-45a1-4f68-ac91-a59ad2d44a2d-fragment.tsv.bgz.tbi` | matching **tabix** index — pycisTopic requires it; no self-indexing |

Two technical reasons this pair wins over the alternates:
- **Barcode consistency.** RNA↔ATAC must share barcodes for the multiome MuData (Stage 4). Taking both
  from one deposit guarantees it; mixing e.g. UCSF-GEX with CELLxGENE-fragments risks a mismatch.
- **No fragment reprocessing.** pycisTopic needs a **bgzip + `.tbi`** fragments file. #2/#3 already are
  that. The Dryad `…atac_fragments.tsv.zip` is a plain 70 GB zip → would need unzip → bgzip → tabix.

**Verified against the actual files (2026-07-14):**
- GEX `.h5ad` = **232,328 cells × 35,477 genes** (whole-transcriptome), **38 samples**.
- Barcode format is **identical** in both: `{Sample_ID}_{16bp}-1` (e.g.
  `ARKFrozen-18-PFC_AAACAGCCAAGGTGCA-1`) — a spot-checked barcode is present in *both* the GEX
  `obs_names` and the fragment file's 4th column. Pairing confirmed, no barcode munging needed.
- **Label column = `obs["Type"]`** (== `obs["Type_updated"]`; they agree on the L2/3 set, differ on
  4,148 cells elsewhere). **`Type == "EN-L2_3-IT"` → 11,928 multiome cells.** (CELLxGENE's standard
  `cell_type` column carries the ontology name "L2/3 intratelencephalic projecting glutamatergic
  neuron"; use the native `Type` for the `EN-L2_3-IT` selection.)

Then in **Stage 1**: read the GEX `.h5ad`, subset `obs["Type"] == "EN-L2_3-IT"` (11,928 cells), emit
the L2/3 barcode list; the fragments stay pooled (all 38 samples) and are filtered to those barcodes
downstream.

### Alternates (fallback only)

- **UCSF portal GEX** (raw counts + native metadata), if the CELLxGENE h5ad lacks a needed field:
  `https://cell.ucsf.edu/snMultiome/source_files/GEX_with_raw_counts.h5ad.gz`
- **Dryad** `10.5061/dryad.2280gb612` (135 GB total) — full deposit, but less convenient for this pipeline:
  - `snMultiome_atlas_atac_fragments.tsv.zip` (70.7 GB) — plain zip, needs bgzip+tabix re-indexing.
  - `snMultiome_atlas_filtered_feature_bc_matrix.h5.zip` (3.4 GB) — 10x h5, metadata is separate.
  - `snMultiome_atlas_Seurat_object.rds` (32.8 GB) — R/Seurat; only if a metadata column is missing above.
- **Raw fastq** (only to reprocess from scratch): NeMO `dat-oiif74w`.
- **MERFISH** spatial (not needed for the GRN): Brain Image Library `10.35077/g.1156`; already on disk
  at `/data/qlyu/data/wang25_human/`.

## Skeleton stages

- **Stage 0 — Acquire data.** Download Wang 2025 snMultiome (see **Data access**): GEX h5ad
  (`GEX_with_raw_counts.h5ad.gz`) + ATAC fragments (Dryad) → `epics/data/wang25/` (gitignored).
- **Stage 1 — Select L2/3.** Subset cells to `EN-L2_3-IT` (Q3). Emit the L2/3 barcode list,
  RNA `.h5ad`, and ATAC fragments restricted to those barcodes.
- **Stage 2 — Human references.** Assemble hg38 assets: genome fasta (on disk), chrom.sizes,
  blacklist, hgnc motif tbl (on disk: `.../motifs-v10-nr.hgnc-m0.00001-o0.0.tbl`), and the
  hg38 cisTarget ranking/score DB (Q2).
- **Stage 3 — pycisTopic.** Peaks (MACS on L2/3 fragments) → cisTopic object → LDA topic
  models → candidate enhancer region sets. (Mirrors astro `11.run_pycistopic.py`.)
- **Stage 4 — SCENIC+ prepare_data.** `scenicplus prepare_GEX_ACC` (build multiome MuData),
  `prepare_menr` + motif enrichment (cistarget/DEM) using the human DB, `search_space`.
- **Stage 5 — GRN inference.** `scenicplus grn_inference {TF_to_gene, region_to_gene, eGRN}`
  → L2/3 eRegulons. (Same CLI verified in the `epics` test run.)
- **Stage 6 — QC / sanity.** eRegulon count, known L2/3 TFs (e.g. CUX2, SATB2, POU3F2/BRN2),
  AUCell, basic plots.
